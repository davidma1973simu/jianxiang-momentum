# 见象（Momentum）开发计划 · Development Plan

> 版本：v0.2（修订版）
> 日期：2026-07-16
> 修订：① 对齐《产品构想》原稿（暖灰米底#F5F0EB+手写体底色、飞行模式物理断联纳入 P1、草原=水墨气泡图与卡通场景融合）；② 语言层语气基调=平等陪伴+共情谦和建议（见语言转换库 v0.2）

## 0. 计划的总原则（来自已确认共识，作为一切验收的底线）

1. **零缺陷暗示**：全程不出现"你有问题 / 你不完整 / 你需要修"。入口让"我很好"成为合法、完整的回答。
2. **工具不是治疗**：5 秒，点击远大于输入，不写、不聊、不分析。治疗暗示你病了，工具不。
3. **镜子不是诊断**：看见大象，不拔、不躲、不解决，就看着。
4. **隐私铁律**：只存枚举，绝不存任何文本；本地优先，"我们不看"可验证。
5. **无 guilt**：不做连胜、不做徽章、不做愧疚式提醒——这是行业流失的根因，从第一天禁止。
6. **双场景一个对象**：草原 + 大象，小大象（日常）与大大象（深度）长在同一片草原，天然是同一产品。
7. **北极星指标**：**周主动触发率**（用户在"大象"出现时主动打开见象的比例），不是 DAU、不是连胜。

## 1. UI/UX 方向（用户明确期待 —— 已列为硬约束）

> 用户原话："我希望能够卡通化、治愈化风格，而且要有草原、大象等等这些隐喻的东西，而且尽量减少哪些按钮、箭头、框架之类的东西。"

### 1.1 风格定位：暖灰底色 + 卡通治愈 / 活着的隐喻
- **暖灰米底（来自《产品构想》原稿）**：基底色 #F5F0EB 系，柔和暖调、接纳不刺激；圆体/手写体标题。这层是"克制、成人、被接住"的底色，必须保留。
- **表层卡通治愈插画**：在暖灰米底之上，草原场景 + 大象角色用圆润笔触、大量留白绘制；远离临床冷感与科技冷感。大象大小/位置/表情编码需求频率与最近状态。
- **融合，非二选一**：底层用《产品构想》的暖灰接纳，表层用 David 最新的"卡通化治愈化"——两者叠合，不是换掉原稿底色。
- **隐喻即界面**：草原 = 首页（你面对的整个世界，他人和自己所在的环境）；大象 = 交互对象（问题 / 挑战 / 麻烦 / 任何让你面对的东西）。没有"菜单栏""工具栏"这类框架，草原本身就是导航。

### 1.2 极简装饰原则（Minimal Chrome Rules）—— 作为每里程碑的**验收标准**
- **无可见按钮**：能"点场景"就不放按钮。点大象 = 有事儿；点晴空 / 脚印 = 今天还行。
- **无箭头引导**：用环境动效、呼吸式过渡表达流向，不画任何箭头。
- **无卡片边框**：用柔和的空间分隔、有机色块、负空间来区隔，不画框、不画分割线。
- **状态靠颜色与天气，不靠徽章与数字**：草原的"天色 / 光线 / 雾"表达你的状态，不用角标、不用计数。
- **动效**：慢、缓动、呼吸感（300–600ms），忌弹跳过度、忌花哨。
- **字体**：圆体无衬线、行高宽松、触控区大（≥ 44pt）。

### 1.3 插画系统（核心资产，必须从 M0 起建）
- **大象情绪态（4–6 种）**：平静 / 探头 / 逼近 / 消散 / 同伴 / 沉睡。
- **草原天气与光态**：黎明 / 晴 / 薄雾 / 雨过天晴 / 星空。
- **道具**：脚印（"今天还行"的轨迹）、"问"的涟漪（触发语言层时的扩散环）、**息屏/震动反馈态**、**灵动岛脚印**（流程中不响铃、只悄悄提示"停一下"，对应飞行模式物理断联）。
- **形式**：矢量 SVG（可随状态驱动、可动画、清晰不失真），统一笔触与配色；可选生成绘感背景图作底层，UI 叠于其上。

### 1.4 无障碍
- 不单靠颜色区分：大象姿态 / 光态 / 动效辅助表达状态；同时控制文字最少化。

### 1.5 声景系统（Soundscape · David 2026-07-17 新增，纳入轨道 A）
> 原话："这个 APP 应该在某些地方有相关的声音，但不要太多喧闹和喧宾夺主。"
- **定位**：点睛的单声，不是背景音乐、不是通知音。服务于"停 / 静 / 向内看"。
- **铁律**：默认极轻、可整体静音；一声即止（≤1.5s+自然衰减），绝不循环 / 铺底 / 连播；只在关键锚点响。
- **声音清单（M1 起建）**：
  - `stop-bowl`：停 / 息屏——低频温暖钵响（听觉版"大象来了"），与轻震动同刻。
  - `touch-ripple`：触动了我——极轻涟漪 / 叶落回应（非奖励"叮"）。
  - `dawn-soft` / `dusk-soft`（P1，默认关）：晨锚点 / 夜复盘的柔和过渡音。
- **绝不做**：通知音、成就 / 连胜音、弹窗"叮"、错误"嗡"、任何催促 / 奖励性音效（听觉版行业 guilt）。
- **实现**：`expo-audio` 加载短音资产，全局静音开关 + 低默认音量；资产存 `assets/sound/`。

## 2. 技术栈（沿用 readme.pdf 规格，按 UI 需求微调）

- **移动端（主交付）**：React Native（Expo）—— 覆盖 iOS / Android。
- **Web（验证与扩展）**：Next.js —— 早期用作可点击原型（验 UI 手感），后期用作"我的草原"Web 视图与多端同步入口。
- **后端 / 存储**：Prisma + SQLite（本地优先）；多端同步阶段可换 Postgres。
- **设计资产**：矢量插画系统（SVG，纳入仓库 `assets/`）；动效用 Lottie / React Native Reanimated。
- **隐私架构**：本地优先，枚举入库，任何文本永不离开设备。

> **关键调整（基于 UI 是核心关切）**：因"治愈卡通 + 极简装饰"实现难度高，M0 先做 **Web 可点击原型**验证"无按钮 / 无框 / 无箭头"的手感与首屏 AHA，再落地 RN 生产版。这能显著降低 UI 返工风险——在写大量功能代码前先把"感觉"钉死。

## 3. 工作流（三条并行轨道）

- **轨道 A · 设计系统**（插画 / 动效 / 设计令牌）—— **必须领先**功能轨道，M0 即产出。
- **轨道 B · 数据与后端**（枚举 schema / 隐私 / 同步）—— 与功能并行，schema 在 M0 冻结。
- **轨道 C · 功能实现**（P0 → P3，按 PRD）。

## 4. 里程碑（精确映射 PRD 的 P0–P3）

### M0 · 基础与视觉语言（2.5–3.5 周）
**目标**：定下"暖灰米底 #F5F0EB + 卡通治愈草原场景 + 极简装饰"的视觉与可点击原型，验证首屏 AHA 不靠文字诊断。
**轨道 A 交付**：
- 设计令牌（柔色板、圆体、间距、动效曲线）
- 插画系统 v0.1（大象 4 态 + 草原 3 光态 + 脚印 / 涟漪）
**轨道 B 交付**：
- 数据 schema v1（枚举模型，见 PRD §5），迁移脚本
**轨道 C 交付**：
- Web 可点击原型：首屏（草原 + 大象，点场景分流，零按钮）→ 大象暂停法 → 语言转换淡显
**验收**：5 名目标用户盲测首屏，能在不读说明下理解"点大象 = 有事儿 / 点晴空 = 还行"，且**无人感到被诊断**。

### M1 · P0 镜子核心（3–4 周）— 对应 PRD P0
**范围**：
- 大象暂停法完整闭环：停（行为阻断）/ 看（点身体部位）/ 问（点需求标签）→ 活人话转换淡显（3 秒）→ 「这句话触动了我」轻触
- 晨间锚点（3 选 1 + 活人话淡显）
- 晚间复盘（滑动条 + 活人话总结）
- 活人话语言转换库 v1（4 需求标签各配，风格铁律：活人话、禁禅意/诗意/大师腔）
- 本地存储（离线可用）
**交付**：iOS / Android 内测包；埋点（周主动触发率、phraseTouched）
**验收**：单次会话中位 5–60s；beta 验证 30 日留存 > 25%；零缺陷暗示审计通过；Minimal Chrome Rules 达标。

### M2 · P1 早晚态 + 数据 + 飞行模式（2–3 周）— 对应 PRD P1
**范围**：
- 早晚界面切换（晨锚点 / 日脚印 / 夜放手）
- `phraseTouched` 落库
- **飞行模式物理断联**（原稿 MVP 必杀技、付费核心卖点）：进暂停法流程自动开系统专注模式、流程结束前屏蔽微信/邮件通知——把"向内求"做成物理级开关
- 基础草原气泡图（横轴时间 / 纵轴需求 / 气泡 = 频率），用天气隐喻呈现，**不用图表边框**
**验收**：用户能"看自己的草原在变"，而不是看一张数据表。

### M3 · P2 三层可视化 + 深度工具（3–4 周）— 对应 PRD P2
**范围**：
- 「我的草原」极简说明（100 字内，方案 B，点破能量 / 磁场大白话，无玄学）
- **四步照象**（全点选，每格"说不清"，第 2→3 格自动翻译 AHA）
- **求助过滤网**（4 点选分流）
- 手写逃生舱（本地便签，标注"只存在你手机上，我们看不到、也不上传"）
- 草原气象可视化（需求模式变化 = 能量 / 磁场的隐喻表达）
**验收**：深度工具月活占比可测；**用户拉杠杆、产品不推**（入口只在暂停法化不开时出现）；全程零文本入库。

### M4 · P3 长期镜像 + 增长种子（3–4 周）— 对应 PRD P3
**范围**：
- AI 周报（含「你的能量和磁场」板块，大白话镜像，不评判）
- 草原变迁（3 个月需求模式地形演变）
- 多端同步
- 惊喜里程碑（自然蹦出，如"你刚才那下是第 20 次面对大象——你其实已经不怕了"，非强制成就）
- 可分享单元（转换卡 / 草原气象截图）—— 增长飞轮钩子
**验收**：留存 cohort 稳定；分享率作为增长信号观测（对应 MRD §7 语言层口碑飞轮）。

### M5 · 打磨与发布验证（2–3 周）
**范围**：
- 性能（冷启动 < 1s）、无障碍、隐私审计（"我们不看"可验证）
- 真实用户 beta：测 30 日留存 > 25%、周主动触发率
- 上线准备（商店素材、治愈卡通风格视觉）
**验收**：达到发布质量门槛；北极星指标在 beta 中显现正向信号。

> **总周期估算**：约 16–21 周至 beta 发布验证。M0 原型不计入功能工期，是 UI 风险的对冲投资。

## 5. 风险与缓解（对齐 MRD §9 假设）

| 风险 / 假设 | 缓解（落在计划何处） |
|------|------|
| 概念太抽象难上手 | M0 原型盲测首屏 AHA；M1 写代码前先验证"不靠文字诊断也能戳中" |
| 极简装饰难落地（无按钮 / 无框 / 无箭头） | 轨道 A 领先；M0 原型即为装饰原则验证；每里程碑按 Minimal Chrome Rules 验收 |
| 不靠连胜 / 提醒能否养成习惯 | 北极星 = 周主动触发率；M1 beta 验证"生活触发反射"假设 |
| 极简不损深度 | M3 四步照象验证"点选也能照出深度" |
| 隐私承诺能否兑现 | 架构本地优先 + 枚举入库；M5 审计"我们不看" |

---

## 6. 完成定义（DoD）

每个里程碑完成 = **功能可用** + **Minimal Chrome Rules 达标**（无按钮 / 无框 / 无箭头验收）+ **零缺陷暗示审计通过** + **埋点就位（仅枚举 / 计数）**。

---

## 7. 全程不做（与 PRD §8 范围外一致）

- ❌ 冥想 / 呼吸计时器
- ❌ 写作日记 / CBT 长记录
- ❌ 诊断 / 替代心理咨询
- ❌ 连胜 / 徽章 / 愧疚式提醒
- ❌ 诱导式课程转化
